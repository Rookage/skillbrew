Agent 能力不是越装越强，是越清楚越强。

很多人真正缺的不是更多工具，而是一个把工具从“看过、收藏过、装过但忘了”变成可管理资产的中间层。SkillBREW 管的是这段混乱。

收藏吃灰刷到必装 MCP，后来找不到来源。
重复安装同类能力换个名字又装一次。
配置漂移Claude Code 和 Codex 路径不同。
黑箱决策AI 说能装，但你不知道它做了什么。
调用遗忘装完以后，不知道怎么触发。

8 步管线，把原料酿成能力。

每一步都能单独跑，中间产物都留在本地目录。它不是一次性魔法，而是一套可审计、可复查、可回滚的工艺。

它真正管三件事。

从素材理解到安装落地之间，SkillBREW 始终保留人工授权门：AI 可以建议，最后由你决定。

判断价值

recommend 先判断值得装、挑着装还是整源跳过，避免“看到清单就全装”。

检查重复

dedup 会扫描本地 Skill、MCP、repo 和历史台账，找出重复和重叠项。

留下台账

record 生成 RECORD、DASHBOARD 和能力索引，让每个能力都有来源、状态和调用方式。

从一条链接开始。

默认不会写入系统配置。先 dry-run 看计划，确认后再加 --approve。

git clone https://github.com/Rookage/skillbrew.git
cd skillbrew
pip install -e .

skillbrew doctor
skillbrew run <url-or-text>
skillbrew verify data/sources/<id>/
skillbrew recommend data/sources/<id>/
skillbrew dedup data/sources/<id>/

# 先看计划
skillbrew install data/sources/<id>/

# 确认后真实安装
skillbrew install data/sources/<id>/ --approve
skillbrew record data/sources/<id>/

已验证范围

  • B站、抖音、YouTube、网页、文本输入源。
  • Skill / MCP / repo / 配置 / Prompt 多形态计划。
  • playwright、filesystem、sequential-thinking、context7、github 等 MCP 种子。
  • fetch-mcp 通过 AI 推断安装链路验证。

仍在打磨

  • memos MCP 暂缓，缺稳定官方 npm 包。
  • MoneyPrinterTurbo 暂缓，依赖额外凭证和运行环境。
  • MCP Marketplace 对接仍在 RFC 阶段。
  • cli / install / installer 模块需要继续拆分。

Claude Code 出生，Codex 也要能住。

SkillBREW 的定位不是某一个运行时的小工具，而是跨 Agent 的能力资产管理层。
